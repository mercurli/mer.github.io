Java面试题-1

2019-02-27 14:01:44 Java

  1. Java面向对象的特征
    • 封装、继承和多态

  1. 多态存在的三个必要条件

    • 要有继承
    • 要有重写
    • 父类引用指向子类对象

  2. 多态的应用场景

    • 同一个接口不同的实现方式

  3. 接口的意义

    • 接口有利于代码规范
    • 接口有利于代码的维护
    • 保证代码的安全与严密

  4. 接口与抽象类的区别

    • 接口是完全抽象的，不存在方法体，而抽象类可以同时包含抽象与非抽象方法
    • 类可以实现多个接口但是只能继承一个抽象类
    • 类可以不实现抽象类和接口声明的所有方法，在这种情况下，类必须是抽象类
    • 抽象类可以在不提供接口方法实现的情况下实现接口
    • 接口中声明的变量默认是final的，而抽象类可包含非final的变量
    • 接口中的方法都必须是public，抽象类则不然
    • 接口不可被实例化，抽象类也不可被实例化，但如果抽象类中包含main方法可以被调用
    • 接口不能有构造方法，抽象类可以

  5. &与&&的区别

    • &是位运算符，表示按位与运算，&&是逻辑运算符，表示逻辑与（and）
    • &&还具有短路的功能

  6. switch是否支持byte与long

    • 支持byte不支持long

  7. ==与eques的区别？equals与hashCode的区别

    • ”==”比较的是两个字符串对象的地址，equals()是比较的两个字符串的具体值。
    • equals方法主要是用来判断从表面上看或者从内容上看，2个对象是不是相等，hashCode相当于是一个对象的编码

  8. Object的公共方法

    • clone()
    • getClass()
    • toString()
    • finalize() 该方法用于释放资源
    • hashCode() 该方法用于哈希查找，可以减少在查找中使用equals的次数，重写了equals方法一般都要重写hashCode方法
    • wait() 调用该方法后当前线程进入睡眠状态
    • notify() 该方法唤醒在该对象上等待的某个线程
    • notifyAll() 该方法唤醒在该对象上等待的所有线程

  9. 堆和栈的区别

    • 在函数中定义的一些基本类型的变量和对象的引用变量都在函数的栈内存中分配
    • 堆内存用来存放有new创建的对象和数组
    • 在堆中产生一个数组或者对象之后，还可以在栈中定义一个特殊变量，让栈中的这个变量的取值等于数组或对象在堆内存中首地址，栈中的这个变量就变成了数组或者对象的引用变量，以后就可以在程序中使用栈中的引用变量来访问堆中的数组或者对象
    • 栈中的变量指向堆内存中的变量，这就是Java的指针

  10. 静态变量与实例变量的区别

    • 静态变量要在前加static关键字修饰
    • 实例变量属于某个对象的变量，必须创建实例才能被使用
    • 静态变量不属于某个实例对象，而是属于类，所以也称为类变量

  11. JVM性能调优

    • JVM将内存划分为New、Tenured(年老)、Perm(永久)，其中New和Perm属于堆内存
    • 线程池
    • 连接池
    • JVM启动参数：调整各代的内存比例和垃圾回收算法，提高吞吐量
    • 程序算法优化：改进程序逻辑算法提高性能
    • 减少使用全局对象和大对象

  12. 反射

    • Java反射就是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个类都能够调用它的任意方法和属性，并且能改变它的属性。
    • 代码可以在运行时装配，无需在组件之间进行源代码的链接，降低代码的耦合度，实现动态代理等

  13. static的作用和用法

    • 用于修饰成员变量，成员方法，而static修饰的成员变量称之为静态变量或类变量
    • static修饰的成员被所有的对象共享
    • static优先于对象存在，因为static的成员随着类的加载就已经存在
    • static修饰的数据是共享数据，对象中的存储的是特有数据

  14. Servlet的生命周期

    • 加载和实例化
    • 初始化
    • 请求处理
    • 服务终止
